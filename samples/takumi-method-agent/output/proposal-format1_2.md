# 匠Method Agent

**匠Methodの知恵を誰もが活用できる世界へ**

～ 生成AIによる匠Methodの自動化・支援エージェント ～

---

## エグゼクティブサマリー

**匠Method Agent** は、ビジネス企画立案のための体系的メソッド「匠Method」を、生成AI（Claude Code）を活用して自動化・支援するツールです。

- 匠Methodの4つのフェーズ（ステークホルダー、価値デザイン、価値分析、要求分析）を対話的に実行
- ファシリテーターの負担軽減、初学者の学習支援、企画品質の向上を実現
- スラッシュコマンドによる直感的な操作で、誰でも匠Methodを活用可能に

# 1. Why - なぜやるのか

---

## 1.1 背景・課題認識

匠Methodは強力なビジネス企画メソッドだが、以下の課題がある：

- **習得に時間がかかる** — 4フェーズの体系的な理解が必要
- **ファシリテーターの負担が大きい** — モデリング作業に多大な労力
- **成果物の作成が手間** — TSV、DrawIO、企画書など多様な形式
- **一貫性の維持が困難** — フェーズ間の整合性チェックが煩雑
- **企画立案の属人化** — 経験者がいないと匠Methodを活用できない

---

## 1.2 ビジョン

> **匠Methodの知恵を誰もが活用できる世界を実現し、すべてのビジネス企画に価値デザインの力を届ける**

匠Methodの本質的な価値 — 「価値からビジネスを考える」という思考法 — を、AIの力で民主化する。

---

## 1.3 私たちの信念

**言葉（キャッチコピー）**:
価値を描く力を、すべての人に

**意味（社会的意義）**:
匠Methodという優れたビジネス企画手法の知恵をAI技術で民主化し、誰もが「価値からビジネスを考える」力を持てる社会を実現する。

---

## 1.3 私たちの信念（続き）

**ストーリー（実現への道筋）**:

1. 匠Methodの各フェーズをAIエージェントとして実装
2. 対話的なモデリング体験を提供し、初学者でも匠Methodを実践可能に
3. ファシリテーターの負担を軽減し、本質的な議論に集中できる環境を構築
4. 匠Methodの普及を加速し、価値デザインの文化を広める

---

# 2. What - 何を提供するのか

---

## 2.1 匠Method Agentとは

Claude Code上で動作する、匠Methodの自動化・支援エージェント。

**入力**: 企画案（`input/idea.md`）
**出力**: 匠Methodの全成果物（TSV、DrawIO、JSON、企画書）

スラッシュコマンドで各フェーズを実行し、匠Methodの4つのモデルを対話的に作成できる。

---

## 2.2 3つのコンセプト

| # | コンセプト | 説明 |
|---|-----------|------|
| 1 | **AIファシリテーション** | 生成AIが匠Methodのファシリテーターとして機能し、各フェーズのモデリングを対話的にガイドする |
| 2 | **段階的モデリング** | 匠Methodの4つのフェーズを段階的に実行し、各フェーズの成果物を自動生成しながらビジネス企画を体系的に構築する |
| 3 | **価値駆動の企画立案** | 常に「価値」を中心に据えた企画立案プロセスを提供し、ステークホルダーへの価値提供を可視化する |

---

## 2.3 匠Methodの4フェーズ（コアモデル）

```
Phase 1              Phase 2              Phase 3              Phase 4
ステークホルダー  →  価値デザイン      →  価値分析          →  要求分析ツリー
モデル               モデル               モデル
─────────────────────────────────────────────────────────────────────
誰に届けるか？      何を届けるか？      どんな価値か？      どう実現するか？
                    （深層的価値）      （表層的価値）
```

- **Phase 1**: ステークホルダーの洗い出しとモデル化
- **Phase 2**: ビジョン・コンセプト・シーズ等の深層的価値をデザイン
- **Phase 3**: 各ステークホルダーへの表層的価値（価値記述）と目的を分析
- **Phase 4**: 業務要求・IT要求・活動に分解し、要求を階層化

---

## 2.4 オプションモデル

コアモデルに加え、以下のオプションモデルも生成可能：

| モデル | 説明 |
|--------|------|
| **ビジネスコンテキストフロー** | ビジネスの全体像をフロー図で可視化 |
| **ゴール記述モデル** | ゴールの詳細を構造化して記述 |
| **因果関係ループ図** | 価値要素間の因果関係を可視化 |
| **MSP計画** | 段階的な実行計画を策定 |
| **企画書** | 成果物を統合した企画書を自動生成 |
| **Value Metrics** | 価値概念のインパクト評価と乖離検証 |

---

## 2.5 私たちの強み（シーズ）

- **匠Methodの体系的知識をAIエージェントとして実装する技術力**
- **Claude Codeのサブエージェント機構を活用した柔軟なフェーズ実行基盤**
- **匠Methodの各フェーズの入出力仕様を熟知したドメイン知識**

# 4. How - どう実現するか

---

## 4.1 目的（価値実現のための施策）

| 目的 | 説明 | 関連コンセプト |
|------|------|----------------|
| 匠Method全フェーズの自動化 | 4フェーズすべてをAIエージェントで自動化し、一貫した成果物生成を実現 | AIファシリテーション |
| 対話型モデリング体験の提供 | ユーザーとの対話を通じてモデリングを進め、学習効果を高める | 段階的モデリング |

| 品質検証・洗練機能 | 各成果物の品質を自動検証し、洗練サイクルを支援 | AIファシリテーション |

---

## 4.2 業務要求

### 匠Method全フェーズの自動化
- 企画案からステークホルダーを自動抽出する機能
- ステークホルダーに基づく価値デザインモデルの自動生成
- ステークホルダー別の価値記述・目的の自動分析
- 目的に基づく業務要求・IT要求・活動の自動展開

### 対話型モデリング体験の提供
- 企画案の対話的作成支援
- 各フェーズの説明コマンド
- サンプル企画の提供

---

## 4.3 業務要求（続き）

### 成果物の多形式出力
- TSVフォーマットでの構造化データ出力
- DrawIO XMLによる視覚的モデル出力

- Markdown形式での企画書自動生成

### 品質検証・洗練機能
- 各フェーズの成果物に対する検証コマンド
- 検証結果に基づく自動洗練コマンド
- Value Metricsによるインパクト評価と乖離検証

---

## 4.4 IT要求・技術要件

# 8. 使用上の注意点

---

## 「仏を作って魂入れず」にならないために

匠Method Agentは強力なツールですが、**AIが生成した成果物をそのまま最終成果物としない**ことが重要です。

### 必ず行うべきこと

1. **AIの出力を鵜呑みにしない** — 生成された成果物は「たたき台」として活用する
2. **チームで議論する** — AIの出力をもとにステークホルダーと対話し、価値を深掘りする
3. **検証・洗練を繰り返す** — `/verify-*` → `/refine-*` のサイクルを活用する
4. **自分の言葉で語れるか確認する** — 企画の本質を自分自身が理解しているか問いかける

---

## AIの限界を理解する

| AIが得意なこと | AIが苦手なこと |
|----------------|----------------|
| 体系的な情報整理 | 現場の暗黙知の把握 |
| 多角的な観点の提示 | ステークホルダーの感情理解 |
| 一貫性のある文書生成 | 企業固有の文化・政治力学 |
| フレームワークの適用 | 創発的なアイデアの飛躍 |

**匠Method Agentは「考える補助輪」であり、「考える代替」ではない。**

# 9. 期待される効果

---

## 9.1 定量的効果
