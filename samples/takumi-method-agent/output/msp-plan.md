# MSP（Minimum Sellable Product）計画

## 概要

### MSPの定義
匠Method Agentの最小販売可能製品として、**個人ユーザーが一人で匠Methodの全4フェーズを完結できるAI支援ツール**を提供する。匠Methodの専門知識がなくても、AIが対話的にガイドしながらビジネス企画のモデリングから企画書生成までを一気通貫で実行できる環境を実現する。

### 対象顧客
**個人ユーザー（起業家、フリーランス、企画担当者）**
- 新規事業や新サービスの企画を体系的に整理したいが、コンサルタントに依頼する予算がない個人
- 匠Methodを知らない、または名前は知っているが実践方法がわからない個人
- 企画書を短時間で作成したいビジネスパーソン

### 提供価値
1. **匠Methodの民主化** - 専門ファシリテーター不在でも、AIガイドにより匠Methodを正しく実践できる
2. **企画プロセスの加速** - 手作業で数日かかるモデリング作業をAIが数分で自動生成する
3. **企画品質の底上げ** - 自動検証により、初心者でも一定水準の企画成果物を作成できる

## 作業計画

### Phase 1: 基盤構築（目安: 4週間）

## リスクと対策

| リスク | 影響度 | 対策 |
|--------|--------|------|
| 匠Methodの暗黙知をAIに正確に実装できない | **高** | 匠Method実践経験13年のビープラウド社内専門家が各フェーズで必ずレビューを実施。正確性検証レビューをPhase1から反復的に行い、早期に精度を確保する |
| LLM APIのコスト増大（全フェーズ自動生成は多数のAPI呼び出しを伴う） | **中** | Phase1段階でAPI呼び出し回数とコストを計測し、バッチ化・キャッシュ戦略を検討。利用料金モデルにAPIコストを適切に反映する |
| 個人ユーザーが匠Methodの前提知識なしに正しく利用できない | **高** | ステップガイダンスとナレッジ配信で作業中に学習できる設計とする。ベータテストで「匠Method未経験者」を必ず含め、理解度を検証する |
| 生成AIの出力品質のばらつき（同じ入力でも結果が異なる） | **中** | 品質自動検証機能で一定基準を下回る出力を検知し、再生成を促す仕組みを実装。プロンプトのテンプレート化と温度パラメータの最適化で安定性を向上させる |
| 競合ツール（AI企画支援サービス）の先行リリース | **中** | 匠Methodという体系的メソッドに基づく点が差別化要因。MSPを12週間以内にリリースし、早期にユーザー基盤と実績を確保する |
| ベータテストでの想定外のユースケース・要望の噴出 | **低** | MSPのスコープを厳格に維持し、追加要望はv1.5/v2.0のバックログに記録。スコープクリープを防止する |

## 次のステップ

MSP完了後の展開計画：

1. **v1.5（MSPリリース後 1〜2ヶ月）**: 品質評価可視化ダッシュボード、非同期バッチ処理機能、進行管理・プロセストラッキング機能、多案件並行管理機能を追加。個人ユーザーのヘビーユース対応を強化する

2. **v2.0（MSPリリース後 3〜4ヶ月）**: チーム共有・共同編集機能、マルチテナント・組織管理機能を追加し、法人チーム向け展開を開始。ROI・投資対効果可視化機能で経営層への訴求力を強化する

3. **v2.5（MSPリリース後 5〜6ヶ月）**: 外部普及・パブリッシュ機能、匠Method普及ダッシュボード機能、AI処理透明性レポート機能を追加。エコシステム拡大と社会普及フェーズに移行する

4. **継続的改善**: ユーザーフィードバック収集・改善データ蓄積機能により、AIモデルの精度向上サイクルを確立。因果関係ループ図の「品質自動検証・フィードバックデータの蓄積 → AI進化速度の向上」の好循環を回し続ける
