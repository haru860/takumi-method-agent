# 実行手順

## 手順1: 知識の読み込み

`Read` で `.claude/skills/takumi-method/value-design.md` を読み込む。

## 手順2: 入力ファイルの読み込み

1. `Read` で `output/value-design.tsv` を読み込む

## 手順3: ビジョンの分析

### 3-1: 良い点の分析（5つ）
以下の観点からビジョンの強みを評価する：
- **明確さ**: 誰に何を提供するかが明確か
- **感情喚起**: 心を動かす表現があるか
- **行動指針**: 意思決定の指針になるか
- **差別化**: 独自性・ユニークさがあるか
- **共感性**: ステークホルダーが自分ごとにできるか
- **未来志向**: ワクワクする未来像があるか
- **簡潔さ**: 短く印象に残るか

### 3-2: 悪い点（改善点）の分析（5つ）
以下の観点から改善すべき点を指摘する：
- **抽象的すぎる**: 具体性が欠けている
- **手段が混入**: ビジョンに手段（How）が入っている
- **自社目線**: 顧客・社会への価値が見えない
- **範囲が曖昧**: 誰のためのビジョンか不明
- **感情に訴えない**: 無機質・官僚的な表現
- **長すぎる**: 記憶に残りにくい
- **必然性の欠如**: なぜ自分たちがやるのか不明

## 手順4: ビジョンを改善するための問いの生成

**重要**: 問いは、手順3で分析した「良い点」と「改善点」に直接紐づけて生成すること。一般的な問いではなく、このビジョン固有の分析結果を踏まえた問いにする。

### 問いの生成方針

5つの問いを以下のように構成する：

1. **改善点1に対応する問い**: 改善点の1つ目を解決するための問い
2. **改善点2に対応する問い**: 改善点の2つ目を解決するための問い
3. **改善点3に対応する問い**: 改善点の3つ目を解決するための問い
4. **良い点を活かす問い**: 良い点をさらに強化・発展させるための問い
5. **良い点と改善点を統合する問い**: 良い点を維持しながら改善点を解消するための問い

### 問いの形式

各問いは以下の形式で出力する：

```
1. 【改善点「○○」に対応】問いの内容...
2. 【改善点「○○」に対応】問いの内容...
3. 【改善点「○○」に対応】問いの内容...
4. 【良い点「○○」を活かす】問いの内容...
5. 【良い点「○○」×改善点「○○」の統合】問いの内容...
```

### 生成のポイント

- 答えやすい問いではなく、**深く考えさせる問い**にする
- 「はい/いいえ」で答えられない**オープンクエスチョン**にする
- **分析した良い点・改善点と明確に対応**させる
- 回答することで**ビジョンが具体化・深化する**問いにする
- 問いの冒頭に【】で対応する良い点・改善点を明示する

## 手順5: 結果の出力

**重要**: 以下の形式を厳密に守ること。特に「ビジョンを深化させるための問い」セクションでは、**必ず対象ビジョンを再掲すること**。これにより、ユーザーが問いを読む際にビジョンを参照しやすくなる。

以下の形式でコンソールに出力する：

```
================================================================================
ビジョンの検証結果
================================================================================

■ 対象ビジョン
「（ビジョンの内容）」

--------------------------------------------------------------------------------
■ 良い点（5つ）
--------------------------------------------------------------------------------
1. 【観点】説明...
2. 【観点】説明...
3. 【観点】説明...
4. 【観点】説明...
5. 【観点】説明...

--------------------------------------------------------------------------------
■ 改善点（5つ）
--------------------------------------------------------------------------------
1. 【観点】説明...
2. 【観点】説明...
3. 【観点】説明...
4. 【観点】説明...
5. 【観点】説明...

================================================================================
ビジョンを深化させるための問い
================================================================================

【必須】■ 対象ビジョン（再掲）
「（ビジョンの内容）」
↑ 【重要】この再掲は省略禁止。必ず出力すること。問いの近くに表示し、ビジョンをみながら問いの答えを考えるため。

以下の問いは、上記の良い点・改善点の分析結果を踏まえて生成しています。

1. 【改善点「○○」に対応】問い...
2. 【改善点「○○」に対応】問い...
3. 【改善点「○○」に対応】問い...
4. 【良い点「○○」を活かす】問い...
5. 【良い点「○○」×改善点「○○」の統合】問い...

================================================================================
```

**注意**: ファイル出力は行わない。すべてコンソール出力のみ。
