## 手順2: IT要求・活動インパクトスコアの分析

インパクトスコアの定義・評価基準は value-metrics.md を参照。

### 2-1: 良い点の分析（5つ）
以下の観点からIT要求・活動インパクトスコアの強みを評価する：
- **スコアの妥当性**: 各IT要求・活動のスコアが実際のインパクトを適切に反映しているか
- **スコア分布の適切さ**: 高スコア・中スコア・低スコアが適切に分散しているか
- **89点（ダントツ）の存在**: 企画の核心となる圧倒的にインパクトの大きい項目が特定されているか
- **業務要求との整合**: IT要求・活動のスコアが関連する業務要求の重要度と整合しているか
- **評価理由の明確さ**: スコアの根拠が論理的に説明されているか
- **IT要求と活動のバランス**: IT要求と活動のスコア配分が適切か
- **目的との整合**: スコアの高い項目が企画の目的達成に直結しているか

### 2-2: 悪い点（改善点）の分析（5つ）
以下の観点から改善すべき点を指摘する：
- **スコアの偏り**: すべて同じようなスコアで優先度が不明確
- **89点（ダントツ）の不在**: 圧倒的にインパクトの大きい項目が特定されていない
- **評価理由の不足**: なぜそのスコアなのか説明が不十分
- **業務要求との乖離**: 重要な業務要求に対応するIT要求・活動のスコアが低い
- **目的との乖離**: 重要な目的に紐づくIT要求・活動のスコアが低い
- **IT要求と活動の不均衡**: 一方に偏ったスコア配分
- **価値概念スコアとの矛盾**: 価値概念のスコアとIT要求・活動のスコアが矛盾
- **実現可能性の無視**: 技術的な実現難易度が考慮されていない

## 手順3: IT要求・活動インパクトスコアを改善するための問いの生成

**重要**: 問いは、手順2で分析した「良い点」と「改善点」に直接紐づけて生成すること。

### 問いの生成方針

5つの問いを以下のように構成する：

1. **改善点1に対応する問い**: 改善点の1つ目を解決するための問い
2. **改善点2に対応する問い**: 改善点の2つ目を解決するための問い
3. **改善点3に対応する問い**: 改善点の3つ目を解決するための問い
4. **良い点を活かす問い**: 良い点をさらに強化・発展させるための問い
5. **良い点と改善点を統合する問い**: 良い点を維持しながら改善点を解消するための問い

### 問いの形式

```
1. 【改善点「○○」に対応】問いの内容...
2. 【改善点「○○」に対応】問いの内容...
3. 【改善点「○○」に対応】問いの内容...
4. 【良い点「○○」を活かす】問いの内容...
5. 【良い点「○○」×改善点「○○」の統合】問いの内容...
```

### 生成のポイント

- 答えやすい問いではなく、**深く考えさせる問い**にする
- 「はい/いいえ」で答えられない**オープンクエスチョン**にする
- **分析した良い点・改善点と明確に対応**させる
- 回答することで**インパクトスコアの妥当性が向上する**問いにする
