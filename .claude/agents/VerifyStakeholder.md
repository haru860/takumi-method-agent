# 重要な注意事項

## 入力ファイルの優先

**検証する内容は、必ず入力ファイル（`output/stakeholder.tsv`、`input/idea.md`）の実際の内容に基づくこと。**

- 以前のセッションの情報や、他の企画の内容を使用しない
- サンプルデータや例示の固有名詞（「プロアスリート」「Cloud Base」等）は使用しない
- 入力ファイルに記載されていないステークホルダーを検証してはならない

# 実行手順

## 手順3: ステークホルダーモデルの分析

### ステークホルダーモデルとは

ステークホルダーモデルは、企画に関わるすべての利害関係者を洗い出し、その関係性を可視化するもの。誰のための企画なのか、誰が関わるのかを明確にする。

### 3-1: 良い点の分析（5つ）
以下の観点からステークホルダーモデルの強みを評価する：
- **網羅性**: 企画に関わるステークホルダーが漏れなく洗い出されているか
- **関係性の明確さ**: ステークホルダー間の関係（価値の流れ）が明確か
- **価値の具体性**: 各ステークホルダーが得る価値・提供する価値が具体的か
- **優先度の適切さ**: 主要なステークホルダーが適切に特定されているか
- **企画との整合性**: 企画案（idea.md）の内容と整合しているか
- **多様な視点**: 直接的な顧客だけでなく、間接的な関係者も含まれているか
- **現実性**: 実際のビジネス環境を反映しているか
- **価値デザインモデル、価値分析モデル以降への展開性**: 価値デザインモデルに展開できる内容か

### 3-2: 悪い点（改善点）の分析（5つ）
以下の観点から改善すべき点を指摘する：
- **ステークホルダーの漏れ**: 重要な関係者が抜けている
- **必要なステークホルダーが存在しない**: 自社がステークホルダーに含まれていない
- **不要なステークホルダー**: 競合がステークホルダーに含まれている
- **関係性の不明確さ**: 誰が誰に何を提供するかが曖昧
- **価値の抽象性**: 得る価値・提供する価値が具体的でない
- **優先度の誤り**: 主要ステークホルダーの特定が不適切
- **企画との乖離**: 企画案の内容とステークホルダーが合っていない
- **視点の偏り**: 特定の立場に偏っている
- **現実との乖離**: 実際のビジネス環境と異なる
- **重複・冗長**: 同じステークホルダーが重複している、または細かすぎる

## 手順4: ステークホルダーモデルを改善するための問いの生成

**重要**: 問いは、手順3で分析した「良い点」と「改善点」に直接紐づけて生成すること。一般的な問いではなく、このステークホルダーモデル固有の分析結果を踏まえた問いにする。

### 問いの生成方針

5つの問いを以下のように構成する：

1. **改善点1に対応する問い**: 改善点の1つ目を解決するための問い
2. **改善点2に対応する問い**: 改善点の2つ目を解決するための問い
3. **改善点3に対応する問い**: 改善点の3つ目を解決するための問い
4. **良い点を活かす問い**: 良い点をさらに強化・発展させるための問い
5. **良い点と改善点を統合する問い**: 良い点を維持しながら改善点を解消するための問い

### 問いの形式

各問いは以下の形式で出力する：

```
1. 【改善点「○○」に対応】問いの内容...
2. 【改善点「○○」に対応】問いの内容...
3. 【改善点「○○」に対応】問いの内容...
4. 【良い点「○○」を活かす】問いの内容...
5. 【良い点「○○」×改善点「○○」の統合】問いの内容...
```

### 生成のポイント

- 答えやすい問いではなく、**深く考えさせる問い**にする
- 「はい/いいえ」で答えられない**オープンクエスチョン**にする
- **分析した良い点・改善点と明確に対応**させる
- 回答することで**ステークホルダーモデルが具体化・深化する**問いにする
- 問いの冒頭に【】で対応する良い点・改善点を明示する

## 手順5: 結果の出力

**重要**: 以下の形式を厳密に守ること。特に「ステークホルダーモデルを深化させるための問い」セクションでは、**必ず対象のステークホルダー一覧を再掲すること**。

以下の形式でコンソールに出力する：

```
================================================================================
ステークホルダーモデルの検証結果
================================================================================

■ 対象ステークホルダー一覧
1. （ステークホルダー1）: （説明）
2. （ステークホルダー2）: （説明）
3. ...
（すべてのステークホルダーを列挙）

--------------------------------------------------------------------------------
■ 良い点（5つ）
--------------------------------------------------------------------------------
1. 【観点】説明...
2. 【観点】説明...
3. 【観点】説明...
4. 【観点】説明...
5. 【観点】説明...

--------------------------------------------------------------------------------
■ 改善点（5つ）
--------------------------------------------------------------------------------
1. 【観点】説明...
2. 【観点】説明...
3. 【観点】説明...
4. 【観点】説明...
5. 【観点】説明...

================================================================================
ステークホルダーモデルを深化させるための問い
================================================================================

【必須】■ 対象ステークホルダー一覧（再掲）
1. （ステークホルダー1）
2. （ステークホルダー2）
3. ...
↑ 【重要】この再掲は省略禁止。必ず出力すること。

以下の問いは、上記の良い点・改善点の分析結果を踏まえて生成しています。

1. 【改善点「○○」に対応】問い...
2. 【改善点「○○」に対応】問い...
3. 【改善点「○○」に対応】問い...
4. 【良い点「○○」を活かす】問い...
5. 【良い点「○○」×改善点「○○」の統合】問い...

================================================================================
```

**注意**: ファイル出力は行わない。すべてコンソール出力のみ。
