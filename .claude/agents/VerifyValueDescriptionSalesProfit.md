# 重要な注意事項

## 入力ファイルの優先

**検証する内容は、必ず入力ファイル（`output/value-analysis-value-description.tsv`、`output/stakeholder.tsv`、`input/idea.md`）の実際の内容に基づくこと。**

- 以前のセッションの情報や、他の企画の内容を使用しない
- サンプルデータや例示の固有名詞（「プロアスリート」「Cloud Base」等）は使用しない
- 入力ファイルに記載されていない価値記述を検証してはならない

# 完了条件

**以下をすべて満たすこと：**

1. 自社（提供者側）の価値記述を特定している

# 実行手順

## 手順1: 知識の読み込み

`Read` で `.claude/skills/takumi-method/value-analysis.md` を読み込む。

## 手順2: 入力ファイルの読み込み

1. `Read` で `input/idea.md` を読み込む（コンテキスト理解用）
2. `Read` で `output/value-analysis-value-description.tsv` を読み込む
   - 存在しない場合はユーザーにPhase 3-1の完了を求める
3. `Read` で `output/value-design.tsv` を読み込む（ビジョン・コンセプト参照用）
4. `Read` で `output/stakeholder.tsv` を読み込む（ステークホルダー情報）
5. `Read` で `input/idea.md` を読み込む（コンテキスト理解用）

## 手順3: 自社の価値記述の特定

ステークホルダーモデルから「自社」に該当するステークホルダーを特定し、その価値記述を抽出する。

**自社を示すキーワード例**:
- 「自社」「当社」「弊社」
- 会社名（idea.mdに記載されている場合）
- 「サービス提供者」「運営者」など

## 手順4: 売上・利益観点での評価

### 4-1: 現状評価

自社の価値記述について、以下の観点で評価する：

**収益性の観点**:
- **価格設定の妥当性**: 提供価値に見合った価格を設定できるか
- **収益源の明確さ**: どこから収益を得るかが明確か
- **収益の持続性**: 一時的ではなく継続的な収益が見込めるか
- **収益規模**: 目標とする売上規模を達成できる価値提案か

**コスト・利益の観点**:
- **コスト構造**: 価値提供にかかるコストは適切か
- **利益率**: 十分な利益率を確保できるか
- **スケーラビリティ**: 事業拡大時にコストが比例して増加しないか

**市場・競争の観点**:
- **市場規模**: ターゲット市場は十分な規模があるか
- **競争優位性**: 価格競争に巻き込まれない差別化があるか
- **顧客獲得コスト**: 顧客を獲得するためのコストは適切か

### 4-2: 課題の特定

現在の価値記述で不十分な点を特定する：
- 収益に直結しない価値記述が多くないか
- 価格に見合う価値が明示されているか
- 顧客が対価を払う理由が明確か

## 手順5: 改善案・追加案の検討

### 5-1: 既存の価値記述の改善案

現在の価値記述を売上・利益向上の観点から改善する案を提示する：
- より収益につながる表現への修正
- 価格設定を正当化する価値の明確化
- 継続利用・リピートにつながる価値の強調

### 5-2: 新たに追加すべき価値記述

売上・利益を向上させるために、新たに追加すべき価値記述を提案する：
- **アップセル・クロスセルにつながる価値**
- **継続利用を促進する価値**
- **紹介・口コミを生む価値**
- **価格プレミアムを正当化する価値**
- **コスト削減につながる価値**
