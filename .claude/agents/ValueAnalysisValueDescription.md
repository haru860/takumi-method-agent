# 重要な注意事項

## 入力ファイルの優先

**生成する内容は、必ず入力ファイル（`input/idea.md`、`output/stakeholder.tsv`、`output/value-design.tsv`）の実際の内容に基づくこと。**

- 以前のセッションの情報や、他の企画の内容を使用しない
- サンプルデータや例示の固有名詞（「プロアスリート」「Cloud Base」等）は使用しない
- 入力ファイルに記載されていない企画やステークホルダー、価値記述を生成してはならない

# 完了条件

**以下をすべて満たすこと：**

1. 重要度「高」のステークホルダーそれぞれに最大5つの価値記述を作成している
2. 重要度「中」のステークホルダーそれぞれに最大3つの価値記述を作成している
3. 重要度「低」のステークホルダーの価値記述は作成していない
4. 各価値記述が「〜で嬉しい」形式になっている

# 実行手順

## 手順1: 知識の読み込み

`Read` で `.claude/skills/takumi-method/value-analysis.md` を読み込む。

## 手順2: 入力ファイルの読み込み

1. `Read` で `input/idea.md` を読み込む
2. `Read` で `output/stakeholder.tsv` を読み込む
3. `Read` で `output/value-design.tsv` を読み込む
4. 企画の背景、ステークホルダー、ビジョン・コンセプトを十分に理解する

## 手順3: 価値記述の作成

手順1で読み込んだ知識（書き方、価値の種類、並び順）に従い作成する。

作成結果として、各価値記述について以下を明確にする：
- ステークホルダー名
- 価値記述名（端的に意味を表す名前）
- 価値記述（「シチュエーション」+「価値を実現する手段」+「〜で嬉しい」形式）
- 価値記述の作成数
    - 重要度「高」のステークホルダーそれぞれ最大5つまで。(価値の高い価値記述を5つ)
    - 重要度「中」のステークホルダーそれぞれ最大3つまで。(価値の高い価値記述を3つ)
    - 重要度「低」のステークホルダーの価値記述は作成しない
