# 重要な注意事項

## 入力ファイルの優先

**検証する内容は、必ず入力ファイル（`output/value-design.tsv`、`input/idea.md`）の実際の内容に基づくこと。**

- 以前のセッションの情報や、他の企画の内容を使用しない
- サンプルデータや例示の固有名詞（「プロアスリート」「Cloud Base」等）は使用しない
- 入力ファイルに記載されていないコンセプトを検証してはならない

# 完了条件

**以下をすべて満たすこと：**

1. コンセプト全体の良い点を5つリストアップしている
2. コンセプト全体の悪い点（改善点）を5つリストアップしている
3. 各コンセプト個別の評価を行っている
4. コンセプトを改善するための問いを5つ生成している
5. すべての結果をコンソールに出力している（ファイル出力は不要）

# 実行手順

## 手順1: 知識の読み込み

`Read` で `.claude/skills/takumi-method/value-design.md` を読み込む。

## 手順2: 入力ファイルの読み込み

1. `Read` で `output/value-design.tsv` を読み込む
   - 存在しない場合はユーザーにPhase 2-2の完了を求める
   - コンセプトが記載されていない場合も同様
2. `Read` で `input/idea.md` を読み込む（コンテキスト理解用）
3. `Read` で `output/stakeholder.tsv` を読み込む（ステークホルダー情報）

## 手順3: コンセプト全体の分析

### 3-1: 良い点の分析（5つ）
以下の観点からコンセプト全体の強みを評価する：
- **ビジョンとの整合性**: ビジョンを実現する道筋になっているか
- **MECE性**: 3つのコンセプトが重複なく、漏れなくビジョンをカバーしているか
- **シーズとの接続**: 自社の強み・技術・資産が活かされているか
- **相乗効果**: 3つが組み合わさることで価値が増幅されるか
- **実現可能性**: 具体的に実行できる内容か
- **差別化**: 競合にはない独自性があるか
- **展開可能性**: Phase 3・4に具体的に展開できるか

### 3-2: 悪い点（改善点）の分析（5つ）
以下の観点から改善すべき点を指摘する：
- **ビジョンとの乖離**: ビジョン実現に貢献しない
- **重複・漏れ**: 3つのコンセプト間で重複があるか、カバーできていない領域がある
- **シーズとの断絶**: 自社の強みが活かされていない
- **抽象的すぎる**: 具体的なアクションにつながらない
- **相乗効果の欠如**: 3つがバラバラで連携していない
- **ステークホルダー不在**: 誰の価値を実現するか不明確
- **実現性の疑問**: 実行が困難、リソースが不足

## 手順4: 各コンセプト個別の評価

3つの各コンセプトについて、以下の観点で簡潔に評価する：

| 評価観点 | 説明 |
|---------|------|
| ビジョン貢献度 | このコンセプトなしにビジョンは実現できるか（◎不可欠 / ○重要 / △あれば良い） |
| シーズ適合度 | 自社の強みで実現できるか（◎自社で完結 / ○一部外部 / △大半外部） |
| ステークホルダー影響 | 誰にどんな価値を届けるか |
| 独自性 | 競合との差別化ポイント |
| 課題・リスク | このコンセプトの弱点や懸念 |

## 手順5: コンセプトを改善するための問いの生成

**重要**: 問いは、手順3・4で分析した「良い点」と「改善点」に直接紐づけて生成すること。一般的な問いではなく、このコンセプト固有の分析結果を踏まえた問いにする。

### 問いの生成方針

5つの問いを以下のように構成する：

1. **改善点1に対応する問い**: 改善点の1つ目を解決するための問い
2. **改善点2に対応する問い**: 改善点の2つ目を解決するための問い
3. **改善点3に対応する問い**: 改善点の3つ目を解決するための問い
4. **良い点を活かす問い**: 良い点をさらに強化・発展させるための問い
5. **良い点と改善点を統合する問い**: 良い点を維持しながら改善点を解消するための問い

### 問いの形式

各問いは以下の形式で出力する：

```
1. 【改善点「○○」に対応】問いの内容...
2. 【改善点「○○」に対応】問いの内容...
3. 【改善点「○○」に対応】問いの内容...
4. 【良い点「○○」を活かす】問いの内容...
5. 【良い点「○○」×改善点「○○」の統合】問いの内容...
```

### 生成のポイント

- 答えやすい問いではなく、**深く考えさせる問い**にする
- 「はい/いいえ」で答えられない**オープンクエスチョン**にする
- **分析した良い点・改善点と明確に対応**させる
- 回答することで**コンセプトが具体化・深化する**問いにする
- 問いの冒頭に【】で対応する良い点・改善点を明示する
