# 重要な注意事項

## 入力ファイルの優先

**生成する内容は、必ず入力ファイル（`input/idea.md`、`output/requirement-tree-business.tsv`）の実際の内容に基づくこと。**

- 以前のセッションの情報や、他の企画の内容を使用しない
- サンプルデータや例示の固有名詞（「プロアスリート」「Cloud Base」等）は使用しない
- 入力ファイルに記載されていない企画や活動を生成してはならない

# 完了条件

1. 活動を10件以上作成している
2. 各活動に説明と関連する業務要求が設定されている

**作成結果は呼び出し元に返却する（TSV出力は呼び出し元が行う）**

# 実行手順

## 手順1: 知識の読み込み
- `Read` で `.claude/skills/takumi-method/requirement-tree.md` を読み込む。

## 手順2: 入力ファイルの読み込み

1. `Read` で `input/idea.md` を読み込む
2. `Read` で `output/requirement-tree-business.tsv` を読み込む
   - 存在しない場合はユーザーにPhase 4-1の完了を求める

## 手順3: 活動の作成

手順1で読み込んだ知識（活動の作成方法）に従い作成する。

作成結果として、各活動について以下を明確にする：
- 活動名
- 説明
- 関連する業務要求

**重要**: 1つの活動が複数の業務要求に関連する場合でも、活動は1件として扱う。同じ活動を複数件として作成しないこと。
**重要**: 「関連する業務要求」は関連度が最も高い業務要求を1つのみ出力すること。カンマ区切り等で複数出力しないこと。

## 手順4: IT要求の確認
1行の「関連する業務要求」に、1件のみ業務要求が出力されている場合、終了。1行に複数の業務要求が出力されているデータがある場合、手順3に戻る。
