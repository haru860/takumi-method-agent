## 手順3: ゴール記述モデルの分析

### ゴール記述モデルとは

ゴール記述モデルは、MSP計画の作業項目を「誰が・何を・いつまでに・どうする」という形で具体化し、評価尺度と目標値を設定するもの。構想を実行可能なアクションプランに落とし込み、進捗を測定可能にする。

### 3-1: 良い点の分析（5つ）
以下の観点からゴール記述モデルの強みを評価する：
- **MSP計画との整合**: MSP計画の作業項目が漏れなくゴール記述に展開されているか
- **具体性**: 「何を」「どうする」が具体的で実行イメージが湧くか
- **測定可能性**: 評価尺度と目標値が定量的・客観的に測定可能か
- **スケジュールの現実性**: 開始時期・終了時期が現実的で、依存関係が考慮されているか
- **役割の明確さ**: 役割名が具体的で、誰が担当するかが明確か
- **網羅性**: MSP計画のすべてのPhase・作業項目がカバーされているか
- **粒度の適切さ**: ゴール記述の粒度が大きすぎず細かすぎず、管理可能か
- **ビジョン・コンセプトとの整合**: 上位の価値設計と一貫したゴール設定か

### 3-2: 悪い点（改善点）の分析（5つ）
以下の観点から改善すべき点を指摘する：
- **MSP計画との乖離**: MSP計画の作業項目が抜けている、または対応していない
- **抽象的すぎる**: 「何を」「どうする」が曖昧で実行イメージが湧かない
- **測定不能**: 評価尺度や目標値が主観的・曖昧で進捗を測定できない
- **スケジュールの非現実性**: 期間が短すぎる・長すぎる、依存関係が無視されている
- **役割の不明確さ**: 担当者が不明確、または一人に集中しすぎている
- **粒度のばらつき**: ゴール記述間で粒度が大きく異なる
- **依存関係の欠如**: 前後のゴール間の依存関係が考慮されていない
- **目標値の甘さ**: 目標値が低すぎて達成しても企画の成功につながらない
- **重複・冗長**: 同じ内容のゴール記述が重複している

## 手順4: ゴール記述モデルを改善するための問いの生成

**重要**: 問いは、手順3で分析した「良い点」と「改善点」に直接紐づけて生成すること。一般的な問いではなく、このゴール記述モデル固有の分析結果を踏まえた問いにする。

### 問いの生成方針

5つの問いを以下のように構成する：

1. **改善点1に対応する問い**: 改善点の1つ目を解決するための問い
2. **改善点2に対応する問い**: 改善点の2つ目を解決するための問い
3. **改善点3に対応する問い**: 改善点の3つ目を解決するための問い
4. **良い点を活かす問い**: 良い点をさらに強化・発展させるための問い
5. **良い点と改善点を統合する問い**: 良い点を維持しながら改善点を解消するための問い

### 問いの形式

各問いは以下の形式で出力する：

```
1. 【改善点「○○」に対応】問いの内容...
2. 【改善点「○○」に対応】問いの内容...
3. 【改善点「○○」に対応】問いの内容...
4. 【良い点「○○」を活かす】問いの内容...
5. 【良い点「○○」×改善点「○○」の統合】問いの内容...
```

### 生成のポイント

- 答えやすい問いではなく、**深く考えさせる問い**にする
- 「はい/いいえ」で答えられない**オープンクエスチョン**にする
- **分析した良い点・改善点と明確に対応**させる
- 回答することで**ゴール記述モデルが具体化・深化する**問いにする
- 問いの冒頭に【】で対応する良い点・改善点を明示する
