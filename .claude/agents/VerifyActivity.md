## 手順3: 活動の分析

### 活動とは

活動は、業務要求を実現するためにITを使わずに人が行う具体的なアクション。「人が何をすべきか」を定義する。業務要求と活動の対応関係が明確であることが重要。

### 3-1: 良い点の分析（5つ）
以下の観点から活動の強みを評価する：
- **業務要求との整合**: 各業務要求に対応する活動が定義されているか
- **具体性**: 活動が具体的で実行可能な形で記述されているか
- **網羅性**: 業務要求の実現に必要な活動が漏れなく洗い出されているか
- **実行可能性**: 現実的に実行できる活動か
- **Phase 2・3との整合**: ビジョン・コンセプト・価値記述と整合しているか
- **担当者の明確さ**: 誰が実行する活動かがイメージできるか
- **成果の測定可能性**: 活動の成果が測定・評価できるか

### 3-2: 悪い点（改善点）の分析（5つ）
以下の観点から改善すべき点を指摘する：
- **業務要求との乖離**: 業務要求と活動がつながっていない
- **抽象的すぎる**: 具体的なアクションイメージが湧かない
- **漏れがある**: 業務要求の実現に必要な活動が欠けている
- **担当者不明確**: 誰が実行するか分からない
- **Phase 2・3との乖離**: ビジョン・コンセプト・価値記述と整合していない
- **実行困難**: リソース・スキル的に非現実的な活動が含まれている
- **重複・冗長**: 同じ内容の活動が重複している
- **成果が不明確**: 活動の結果として何が得られるか分からない
- **IT要求との区別が曖昧**: IT要求と活動の境界が不明確

## 手順4: 活動を改善するための問いの生成

**重要**: 問いは、手順3で分析した「良い点」と「改善点」に直接紐づけて生成すること。一般的な問いではなく、この活動固有の分析結果を踏まえた問いにする。

### 問いの生成方針

5つの問いを以下のように構成する：

1. **改善点1に対応する問い**: 改善点の1つ目を解決するための問い
2. **改善点2に対応する問い**: 改善点の2つ目を解決するための問い
3. **改善点3に対応する問い**: 改善点の3つ目を解決するための問い
4. **良い点を活かす問い**: 良い点をさらに強化・発展させるための問い
5. **良い点と改善点を統合する問い**: 良い点を維持しながら改善点を解消するための問い

### 問いの形式

各問いは以下の形式で出力する：

```
1. 【改善点「○○」に対応】問いの内容...
2. 【改善点「○○」に対応】問いの内容...
3. 【改善点「○○」に対応】問いの内容...
4. 【良い点「○○」を活かす】問いの内容...
5. 【良い点「○○」×改善点「○○」の統合】問いの内容...
```

### 生成のポイント

- 答えやすい問いではなく、**深く考えさせる問い**にする
- 「はい/いいえ」で答えられない**オープンクエスチョン**にする
- **分析した良い点・改善点と明確に対応**させる
- 回答することで**活動が具体化・深化する**問いにする
- 問いの冒頭に【】で対応する良い点・改善点を明示する
