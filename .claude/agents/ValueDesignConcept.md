# 重要な注意事項

## 入力ファイルの優先

**生成する内容は、必ず入力ファイル（`input/idea.md`、`output/stakeholder.tsv`、`output/value-design.tsv`）の実際の内容に基づくこと。**

- 以前のセッションの情報や、他の企画の内容を使用しない
- サンプルデータや例示の固有名詞（「プロアスリート」「Cloud Base」等）は使用しない
- 入力ファイルに記載されていない企画やコンセプトを生成してはならない

# 完了条件

**以下をすべて満たすこと：**

1. コンセプトを3つ作成している
2. 各コンセプトに名前と説明がある
3. コンセプトがビジョンと整合している

**作成結果は呼び出し元に返却する（TSV出力は呼び出し元が行う）**

# 実行手順

## 手順1: 知識の読み込み

`Read` で `.claude/skills/takumi-method/value-design.md` を読み込む。

## 手順2: 入力ファイルの読み込み

1. `Read` で `input/idea.md` を読み込む
2. `Read` で `output/stakeholder.tsv` を読み込む
   - 存在しない場合はユーザーにステークホルダーモデル（Phase 1）の完了を求める
3. `Read` で `output/value-design.tsv` を読み込む
   - ビジョンを確認し、コンセプト作成の基盤とする

## 手順3: コンセプトの作成

手順1で読み込んだ知識（コンセプトの作成方法）に従い作成する。
- 思考のプロセスをコンソールに出力する
- ビジョンを実現するための3つの柱となるコンセプトを作成

作成結果として、以下を明確にする：
- コンセプト1の名前、コンセプト1
- コンセプト2の名前、コンセプト2
- コンセプト3の名前、コンセプト3
