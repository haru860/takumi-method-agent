# 実行手順

   - 存在しない場合はユーザーにPhase 1の完了を求める

## 手順3: ビジョンの作成
思考のプロセスをコンソールに出力する

### 3-1: 問いを立てる
以下の問いに答えてビジョンの核を見つける：
- この企画がなかったら、誰が困るのか？
- 10年後、何が実現していたら最高か？
- なぜ自分たちがやる必要があるのか？

### 3-2: 深層的価値を掘り下げる
ステークホルダーの課題から深層的価値を見つける：
- 「それが実現すると、何が嬉しいですか？」
- 「なぜそれが重要なのですか？」
- 「本当に欲しいものは何ですか？」

### 3-3: ビジョンを言語化する
**基本形**: 「[誰が] + [どうなる] + [世界/社会]」

以下を意識する：
- 主語を明確に（誰のためか）
- 動詞で締める（創る、届ける、実現する）
- 感情を動かす言葉を選ぶ

### 3-4: ビジョンの検証
魅力的なビジョンの4条件で検証する：
- **必然性**: なぜ自分たちがやるのか
- **明確さ**: 10秒で説明できるか
- **行動を生む**: 意思決定の指針になるか
- **共感を生む**: チームや顧客が自分ごとにできるか

### 避けるべきパターン
- 抽象的すぎる（「より良い未来を創る」）
- 手段が入っている（「AIで業務を効率化する」）
- 自社目線すぎる（「業界No.1になる」）

作成結果として、以下を明確にする：
- ビジョン（1文）
- ビジョンが4条件をどう満たしているかの説明
