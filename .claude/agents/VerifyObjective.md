## 手順3: 目的の分析

### 目的とは

目的は、価値記述を実現するために達成すべき具体的なゴール。各価値記述に対して「何を達成すれば価値が実現されるか」を定義する。Phase 4の業務要求・IT要求の起点となる。

### 3-1: 良い点の分析（5つ）
以下の観点から目的の強みを評価する：
- **価値記述との整合**: 各価値記述に対応する目的が定義されているか
- **具体性**: 目的が具体的で達成可能な形で記述されているか
- **測定可能性**: 目的の達成度を測定できるか
- **ステークホルダー紐付け**: 誰のための目的かが明確か
- **価値デザインモデル(Phase 2)との整合**: ビジョン・コンセプトと整合しているか
- **要求分析ツリー(Phase 4)への展開性**: 業務要求・IT要求に展開できる内容か
- **優先度の明確さ**: 目的間の優先度が適切か
- **実現可能性**: 現実的に達成できる目的か

### 3-2: 悪い点（改善点）の分析（5つ）
以下の観点から改善すべき点を指摘する：
- **価値記述との乖離**: 価値記述と目的がつながっていない
- **抽象的すぎる**: 具体的な達成イメージが湧かない
- **測定困難**: 達成したかどうか判断できない
- **ステークホルダー不明確**: 誰のための目的か分からない
- **価値デザインモデル(Phase 2)との乖離**: ビジョン・コンセプトと整合していない
- **展開困難**: Phase 4に具体的に展開できない
- **優先度の不明確さ**: どの目的が重要か分からない
- **実現困難**: 達成が非現実的な目的が含まれている

## 手順4: 目的を改善するための問いの生成

**重要**: 問いは、手順3で分析した「良い点」と「改善点」に直接紐づけて生成すること。一般的な問いではなく、この目的固有の分析結果を踏まえた問いにする。

### 問いの生成方針

5つの問いを以下のように構成する：

1. **改善点1に対応する問い**: 改善点の1つ目を解決するための問い
2. **改善点2に対応する問い**: 改善点の2つ目を解決するための問い
3. **改善点3に対応する問い**: 改善点の3つ目を解決するための問い
4. **良い点を活かす問い**: 良い点をさらに強化・発展させるための問い
5. **良い点と改善点を統合する問い**: 良い点を維持しながら改善点を解消するための問い

### 問いの形式

各問いは以下の形式で出力する：

```
1. 【改善点「○○」に対応】問いの内容...
2. 【改善点「○○」に対応】問いの内容...
3. 【改善点「○○」に対応】問いの内容...
4. 【良い点「○○」を活かす】問いの内容...
5. 【良い点「○○」×改善点「○○」の統合】問いの内容...
```

### 生成のポイント

- 答えやすい問いではなく、**深く考えさせる問い**にする
- 「はい/いいえ」で答えられない**オープンクエスチョン**にする
- **分析した良い点・改善点と明確に対応**させる
- 回答することで**目的が具体化・深化する**問いにする
- 問いの冒頭に【】で対応する良い点・改善点を明示する
