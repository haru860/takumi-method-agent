## 手順3: IT要求の分析

### IT要求とは

IT要求は、業務要求を実現するために必要なITシステムの要求事項。「ITで何を実現すべきか」を定義する。業務要求とIT要求の対応関係が明確であることが重要。

### 3-1: 良い点の分析（5つ）
以下の観点からIT要求の強みを評価する：
- **業務要求との整合**: 各業務要求に対応するIT要求が定義されているか
- **具体性**: IT要求が具体的で実装可能な形で記述されているか
- **網羅性**: 業務要求の実現に必要なIT要求が漏れなく洗い出されているか
- **実現可能性**: 技術的に実現可能なIT要求か
- **Phase 2・3との整合**: ビジョン・コンセプト・価値記述と整合しているか
- **ユーザー視点**: ステークホルダーにとって価値のあるIT要求か
- **優先度の明確さ**: IT要求間の優先度が適切か

### 3-2: 悪い点（改善点）の分析（5つ）
以下の観点から改善すべき点を指摘する：
- **業務要求との乖離**: 業務要求とIT要求がつながっていない
- **抽象的すぎる**: 具体的なシステムイメージが湧かない
- **漏れがある**: 業務要求の実現に必要なIT要求が欠けている
- **技術偏重**: 技術的な実装に偏り、ビジネス価値が不明確
- **Phase 2・3との乖離**: ビジョン・コンセプト・価値記述と整合していない
- **実現困難**: 技術的・コスト的に非現実的なIT要求が含まれている
- **重複・冗長**: 同じ内容のIT要求が重複している
- **優先度の不明確さ**: どのIT要求が重要か分からない

## 手順4: IT要求を改善するための問いの生成

**重要**: 問いは、手順3で分析した「良い点」と「改善点」に直接紐づけて生成すること。一般的な問いではなく、このIT要求固有の分析結果を踏まえた問いにする。

### 問いの生成方針

5つの問いを以下のように構成する：

1. **改善点1に対応する問い**: 改善点の1つ目を解決するための問い
2. **改善点2に対応する問い**: 改善点の2つ目を解決するための問い
3. **改善点3に対応する問い**: 改善点の3つ目を解決するための問い
4. **良い点を活かす問い**: 良い点をさらに強化・発展させるための問い
5. **良い点と改善点を統合する問い**: 良い点を維持しながら改善点を解消するための問い

### 問いの形式

各問いは以下の形式で出力する：

```
1. 【改善点「○○」に対応】問いの内容...
2. 【改善点「○○」に対応】問いの内容...
3. 【改善点「○○」に対応】問いの内容...
4. 【良い点「○○」を活かす】問いの内容...
5. 【良い点「○○」×改善点「○○」の統合】問いの内容...
```

### 生成のポイント

- 答えやすい問いではなく、**深く考えさせる問い**にする
- 「はい/いいえ」で答えられない**オープンクエスチョン**にする
- **分析した良い点・改善点と明確に対応**させる
- 回答することで**IT要求が具体化・深化する**問いにする
- 問いの冒頭に【】で対応する良い点・改善点を明示する
