# 価値デザインモデル(value-design)

## 価値デザインモデル

企画が実現する**深層的価値**（本質的な価値）をデザインする。「意」のモデル。

「なぜこの企画をやるのか」の本質を明確にする。

「ビジョン」「３つのコンセプト」「言葉」「意味」「デザイン」「ストーリー」という要素で表現する。これらの要素を総合的に調和することにより、自分たちの力になるような「心地よい価値を演出する」ことを目指す。（書籍「匠Method:〜新たな価値観でプロジェクトをデザインするために〜」第4章より引用）

### 価値デザインモデルの構成要素

#### 1. ビジョン
- 企画が目指す**理想の未来像**
- 「こうなったらいいな」という世界観
- 抽象的でも良いが、共感を呼ぶものにする

#### 2. コンセプト
- ビジョンを実現するための**中心的なアイデア**
- 企画の核となる考え方
- ビジョンと具体的な手段をつなぐ

#### 3. シーズ（種）
- 企画を実現するための**技術・強み・資産**
- 自社が持っているもの、活用できるもの
- 例: 技術力、ノウハウ、ネットワーク、データ

#### 4. 深層的価値
- ステークホルダーにとっての**本質的な価値**
- 表面的なメリットではなく、根源的な価値
- 「本当に欲しいもの」「本当に解決したいこと」

#### 5. 言葉（キャッチコピー）
- ビジョンを端的に表現する**印象的なフレーズ**
- 覚えやすく、共感を呼ぶもの
- 企画の本質を一言で伝える

#### 6. 意味
- 価値デザインモデル全体を文章で説明する

#### 7. ストーリー
- ビジョンを実現するための**ステップ・道筋**
- 現状から理想の未来への変化の物語
- 箇条書きで具体的なステップを示す

### 8. デザイン
- ビジョンを視覚的に表現する**デザインコンセプト**
- 色、形、イメージの方向性
- ロゴやビジュアルの基本方針

## ビジョン

### ビジョンとは
ビジョンとは、企画や事業が実現した結果として立ち現れる「理想の状態」を言語化したもの。

それは数値目標ではなく、関係者や社会にどのような変化が起き、どのような価値が当たり前になっているかという“到達後の世界像”を示す。

ビジョンは、企画の出発点であると同時に、意思決定の最上位に位置する判断基準である。

目標や施策は変わっても、ビジョンは変わりません。企画の一貫性を支え続ける「北極星」となる概念。


### 魅力的なビジョンの条件

#### 1. 必然性がある
- 社会的意義と自社の存在理由がつながっている
- 三方良し、八方良しの社会的価値を体現する
- 強い原体験や動機がある

#### 2. 明確である
- シンプルで覚えやすい（10秒で説明できる）
- 抽象的すぎず、具体的なイメージが湧く
- 例: 「誰もが3クリックで必要な医療情報を得られる社会をつくる」

#### 3. 行動を生み出す
- ビジョンに基づいて意思決定ができる
- 目標や戦略に落とし込める

#### 4. 共感を生む
- チームメンバーが「これは自分たちのビジョンだ」と思える
- 顧客や社会とつながっている

---

### ビジョンの構造パターン

#### 基本形
「[誰が] + [どうなる] + [世界/社会]」

#### 良いビジョンの例
- 「誰もが3クリックで必要な医療情報を得られる社会をつくる」
- 「すべての中小企業が財務の不安なく事業に集中できる世界」
- 「誰もがAIを味方に、ビジネスの想いを形にできる世界を創る」

#### 避けるべきパターン
| パターン | 問題点 | 例 |
|---------|--------|-----|
| 抽象的すぎる | 具体的なイメージが湧かない | 「より良い未来を創る」 |
| 手段が入っている | ビジョンではなく戦略になっている | 「AIで業務を効率化する」 |
| 自社目線すぎる | 顧客や社会の視点がない | 「業界No.1になる」 |

---

### ビジョン作成の手順

#### Step 1: 問いを立てる
以下の問いに答えることで、ビジョンの核を見つける：
- この企画がなかったら、誰が困るのか？
- 10年後、何が実現していたら最高か？
- なぜ自分たちがやる必要があるのか？

#### Step 2: 深層的価値を掘り下げる
表層的な価値から深層的価値へ掘り下げる：

| 表層的価値 | 深層的価値 |
|-----------|-----------|
| コスト削減 | 経営の安定・安心 |
| 時間短縮 | 本来やりたいことに集中 |
| 便利になる | ストレスからの解放 |
| スキルアップ | 自信を持って仕事ができる |
| 効率化 | 創造的な仕事に時間を使える |

**掘り下げの質問**:
1. 「それが実現すると、何が嬉しいですか？」
2. 「なぜそれが重要なのですか？」
3. 「本当に欲しいものは何ですか？」

#### Step 3: 言葉を磨く
- **主語を明確に**: 誰のためのビジョンか
- **動詞で締める**: 創る、届ける、実現する、つくる
- **感情を動かす言葉を選ぶ**: 共感を呼ぶ表現

---

### ビジョン作成のよくある失敗と対処法

| 失敗パターン | 問題点 | 対処法 |
|-------------|--------|--------|
| スローガン止まり | 行動につながらない | 具体的な状態を描く |
| 機能の羅列 | 共感を生まない | 「なぜ」を問い続ける |
| 自社都合 | 顧客視点がない | ステークホルダーの課題から出発 |
| 壮大すぎ | 現実味がない | 自社の強み（シーズ）と接続する |

- [ ] チームメンバーに話したとき、目が輝くか？

- [ ] ステークホルダーの課題解決につながっているか？

## コンセプト
### コンセプトとは
コンセプトとは、**ビジョンという「目的地」に到達するための「3本の道」**であり、構想のこと。

### コンセプトの重要なポイント：
- ビジョン（実現したい世界）から**逆算**して導く
- 自社のシーズ（強み・技術・資産）に**根ざしている**
- ステークホルダーの深層的価値を**実現する手段**になっている

### なぜ「3つ」なのか

| 理由 | 説明 |
|------|------|
| **戦略の多面性** | ビジョンは複合的な価値を含むため、1つの戦略では実現できない。異なる角度からアプローチする必要がある |
| **シーズの活用** | 自社が持つ複数の強み・資産を活かすには、複数の戦略軸が必要 |
| **リスク分散** | 1つの戦略が失敗しても、他の戦略で補完できる |
| **相乗効果** | 3つの戦略が絡み合うことで、単独では生まれない価値が創出される |

### コンセプト導出の5ステップ

#### Step 1: ビジョンを「価値の束」として分解する

ビジョンには複数の価値が含まれている。まず、ビジョンを構成する価値要素を分解する。

**例**: ビジョン「すべての人が健康で自分らしく生きられる社会」

| 価値要素 | 内容 |
|---------|------|
| 価値A | 健康状態の維持・改善 |
| 価値B | 自分らしさの追求・実現 |
| 価値C | すべての人（格差のない包摂） |

#### Step 2: シーズ（自社の強み）を棚卸しする

コンセプトは「やりたいこと」だけでなく「できること」に根ざす必要がある。

**棚卸しの問い**:
- 自社が持つ技術・ノウハウは何か？
- 自社が持つネットワーク・関係性は何か？
- 自社が持つデータ・資産は何か？
- 自社だからこそできることは何か？

#### Step 3: 「価値要素 × シーズ」のマトリクスで交点を探す

ビジョンの価値要素と、自社のシーズを掛け合わせ、**実現可能かつインパクトのある戦略**を見つける。

|  | シーズ1 | シーズ2 | シーズ3 |
|--|--------|--------|--------|
| **価値A** | ◎ | ○ | △ |
| **価値B** | ○ | ◎ | ○ |
| **価値C** | △ | ○ | ◎ |

**◎の交点**から、有力なコンセプト候補が見えてくる。

##### 「実現可能かつインパクトのある戦略」の判断基準

**実現可能性の基準**

| 基準 | 判断の問い | 高◎ / 中○ / 低△ |
|------|-----------|-----------------|
| **シーズとの適合度** | 自社の強み・技術・資産で実現できるか？ | ◎自社で完結 ○一部外部 △大半外部 |
| **リソースの充足度** | 必要な人材・資金・時間は確保できるか？ | ◎十分 ○調達可能 △不足 |
| **技術的成熟度** | 技術は実用段階にあるか？実績はあるか？ | ◎実績あり ○検証済み △未検証 |
| **組織的実行力** | 自社の組織文化・体制で実行できるか？ | ◎得意領域 ○対応可能 △苦手領域 |
| **外部環境の制約** | 法規制・業界慣行・競合状況に阻まれないか？ | ◎障壁なし ○対応可能 △重大障壁 |

**インパクトの基準**

| 基準 | 判断の問い | 高◎ / 中○ / 低△ |
|------|-----------|-----------------|
| **ビジョン貢献度** | この戦略なしにビジョンは実現できるか？ | ◎不可欠 ○重要 △あれば良い |
| **ステークホルダー影響範囲** | 何人の、どのステークホルダーに価値を届けられるか？ | ◎広範囲 ○主要層 △限定的 |
| **課題解決の深さ** | 根本的な課題を解決するか？表層的な改善か？ | ◎根本解決 ○大幅改善 △部分改善 |
| **差別化・独自性** | 競合にない独自の価値を生み出せるか？ | ◎唯一無二 ○優位性あり △横並び |
| **持続性・発展性** | 一時的な効果か？長期的に価値を生み続けるか？ | ◎発展可能 ○持続可能 △一時的 |
| **波及効果** | 他のコンセプトや事業への相乗効果があるか？ | ◎大きい ○ある △なし |

**総合判断マトリクス**

実現可能性とインパクトの2軸で評価し、優先順位を決める。

| 領域 | 実現可能性 | インパクト | 判断 |
|------|-----------|-----------|------|
| **最優先** | 高 | 高 | コンセプトとして採用 |
| **検討** | 低 | 高 | 実現性を高める方法を探る（パートナー連携等） |
| **補助的** | 高 | 低 | 他コンセプトの補強として活用 |
| **見送り** | 低 | 低 | 優先度を下げる |

#### Step 4: 3つのコンセプトをMECEに整理する

交点から見えた候補を、**重複なく・漏れなく**3つに整理する。

整理の視点：
- **対象軸**: 誰に対する戦略か（顧客層・ステークホルダー別）
- **価値軸**: どんな価値を提供するか（機能・体験・関係性）
- **実現軸**: どうやって実現するか（技術・仕組み・人）

#### Step 5: 後続Phaseへの展開を確認する

コンセプトは、Phase 3（価値分析）・Phase 4（要求分析）に展開される。

| コンセプト | → Phase 3（目的・価値記述） | → Phase 4（業務要求・IT要求） |
|-----------|---------------------------|------------------------------|
| コンセプト1 | ステークホルダーの価値 | 具体的な要求 |
| コンセプト2 | ステークホルダーの価値 | 具体的な要求 |
| コンセプト3 | ステークホルダーの価値 | 具体的な要求 |

**後続に展開できないコンセプトは、抽象的すぎるか、実現性が低い**ことを示す。

### コンセプト作成のコツ

1. **MECEを意識する**: 3つが重複せず、かつビジョン実現に必要な領域をカバーする
2. **名前を端的につける**: 各コンセプトは「〇〇の△△」のように短いフレーズで表現
3. **シーズと接続する**: 自社の強み・技術・資産が活かせる戦略にする
4. **独立して成立する**: 各コンセプトが単独でも意味を持ち、価値を説明できる
5. **相乗効果を生む**: 3つが組み合わさることで、ビジョン実現の確度が上がる

### コンセプトの検証チェックリスト

- [ ] 各コンセプトはビジョンのどの価値要素を実現するか説明できるか？
- [ ] 各コンセプトは自社のシーズに根ざしているか？
- [ ] 3つのコンセプトでビジョン全体をカバーしているか（MECE）？
- [ ] 各コンセプトはPhase 1のステークホルダーの価値につながるか？
- [ ] 各コンセプトはPhase 3・4に具体的に展開できるか？
- [ ] 3つが相乗効果を生む関係になっているか？

#### コンセプトの例
- 「AIが経理業務を自動化し、経営者に時間を返す」
- 「マッチングで最適な働き方を見つける」

## 言葉（キャッチコピー）

### 言葉（キャッチコピー）とは
**言葉は、ビジョンとコンセプトを「伝える力」に変換するもの**。

| 役割 | 説明 |
|------|------|
| **ビジョンの結晶化** | 理想の未来像を「一言」に凝縮する |
| **共感の入口** | ステークホルダーが最初に触れるメッセージ |
| **意思決定の軸** | 迷ったときに立ち返る言葉 |
| **意味・ストーリーへの橋渡し** | 言葉から意味が広がり、ストーリーが生まれる |

### 言葉の作成手順

#### Step 1: ビジョンから「核となる価値」を抽出する

ビジョンを読み、以下を問う：
- このビジョンで最も伝えたい価値は何か？
- ステークホルダーが「これは自分のことだ」と感じるポイントはどこか？

**例**：
```
ビジョン: 「すべての人が健康で自分らしく生きられる社会」
核となる価値: 「自分らしさ」「すべての人」
```

#### Step 2: コンセプト（3つ）から「行動・変化」を抽出する

3つのコンセプトが示す方向性を言葉に反映させる：
- コンセプトはビジョン実現の「手段」
- 言葉はその手段を「約束」として表現する

#### Step 3: シーズ（自社の強み）を「らしさ」として織り込む

**重要**: 言葉にも「なぜ自分たちがやるのか」が滲み出るべき。

- 自社のシーズがあるからこそ言える言葉にする
- 競合には言えない独自性を表現する

#### Step 4: 複数案を作成し、検証する

最低5案は作成し、以下の基準で絞り込む。

### 言葉の品質基準

| 基準 | 問い |
|------|------|
| **ビジョン適合** | ビジョンの本質を表現しているか？ |
| **コンセプト反映** | 3つのコンセプトの方向性と矛盾しないか？ |
| **シーズ接続** | 自社の強みがあるからこそ言える言葉か？ |
| **ステークホルダー共感** | 対象者が「自分ごと」と感じるか？ |
| **ストーリーへの発展性** | この言葉から「物語」が生まれるか？ |

### 言葉の構造パターン

| パターン | 特徴 | 適するケース | 例 |
|---------|------|-------------|-----|
| **ビジョン凝縮型** | ビジョンを短縮して言葉にする | ビジョンが明確なとき | 「誰もがAIを味方に」 |
| **約束型** | ステークホルダーへの約束 | コンセプトが明確なとき | 「あなたの想いを、形に」 |
| **変化表現型** | Before→Afterを表現 | 課題解決型の企画 | 「不安から自信へ」 |
| **問いかけ型** | ステークホルダーに問う | 共感を重視するとき | 「本当に大切なことは何か」 |
| **造語型** | 新しい概念を作る | 差別化を強調するとき | 独自の造語 |

### 言葉作成のよくある失敗と対処法

| 失敗パターン | 問題点 | 対処法 |
|-------------|--------|--------|
| **機能説明になる** | 「AIで効率化」→共感を生まない | 「効率化した結果、何が得られるか」を表現 |
| **抽象的すぎる** | 「より良い未来へ」→差別化できない | ビジョンの具体的な価値を入れる |
| **自社目線** | 「No.1を目指す」→顧客に関係ない | ステークホルダーが主語の表現に |
| **コンセプトと乖離** | コンセプトが示す方向と言葉が矛盾 | コンセプトを読み直し整合を取る |
| **シーズと断絶** | 誰でも言える言葉になる | 自社の強みを反映した表現に |

### 言葉の検証チェックリスト

- [ ] ビジョンを知らない人がこの言葉を聞いて、ビジョンの方向性を感じ取れるか？
- [ ] 3つのコンセプトと矛盾していないか？
- [ ] 自社のシーズ（強み）があるからこそ言える言葉か？
- [ ] ステークホルダーが聞いて「自分のことだ」と感じるか？
- [ ] この言葉から「ストーリー」「デザイン」が展開できるか？
- [ ] 競合には言えない独自性があるか？
- [ ] 5年後も色褪せないか？

## 意味

### 意味の位置づけ

意味は価値デザインモデルの構成要素の一つで、**価値デザインモデル全体を文章で説明する**ものです。

| 観点 | 説明 |
|-----|------|
| 位置づけ | 価値デザインモデルの要素をつなぐ「意味の文脈」 |
| 役割 | ビジョン・コンセプト・言葉の「なぜ」を伝える |
| 対象 | ステークホルダー全体（特に意思決定者） |
| 入力 | ビジョン、コンセプト（3つ）、言葉、シーズ |

### 意味の役割

| 役割 | 説明 |
|------|------|
| **背景の説明** | なぜこの企画が必要なのか、社会的背景・問題意識を伝える |
| **価値の統合** | ビジョン・コンセプト・言葉がどう結びつくかを文章で示す |
| **動機の共有** | 企画者の思い・原体験を伝え、共感を引き出す |
| **理解の促進** | 抽象的なビジョンやコンセプトを、具体的な文脈で理解しやすくする |

### 意味の作成方法

**Step 1: 企画の背景を整理する**
- なぜこの企画が必要か？（社会的背景・課題）
- 企画者自身がこの企画に取り組む理由・動機は？

**Step 2: ビジョン・コンセプト・言葉を文脈でつなぐ**
- ビジョンが目指す未来像を起点に
- 3つのコンセプトがどのようにビジョンを実現するかを説明
- 言葉（キャッチコピー）が何を象徴しているかを明確に

**Step 3: ステークホルダーにとっての意味を記述する**
- この企画が実現すると、ステークホルダーにとって何が変わるか
- なぜステークホルダーにとって重要なのか

### 良い意味・悪い意味の例

**良い例**:
| 特徴 | 説明 |
|------|------|
| 背景と企画が結びついている | 社会的課題から企画の必要性が伝わる |
| ビジョン・コンセプトと整合 | 意味を読めば価値デザインの全体像が理解できる |
| 企画者の思いが感じられる | 「なぜ自分たちがやるのか」が伝わる |
| ステークホルダー視点がある | 読み手が「自分に関係がある」と感じる |

**悪い例**:
| 問題パターン | 問題点 |
|-------------|--------|
| 機能の説明に終始 | 「何をするか」はあるが「なぜやるか」がない |
| ビジョンの繰り返し | ビジョンをそのまま長文にしただけで新しい情報がない |
| 自社都合の主張 | ステークホルダー視点がなく、一方的な主張になっている |
| 抽象的すぎる | 具体的なイメージが湧かない |

### 意味の検証チェックリスト

- [ ] 企画の背景・社会的意義が明確に記述されているか？
- [ ] ビジョン・コンセプト・言葉と整合しているか？
- [ ] 企画者の動機・思いが感じられるか？
- [ ] ステークホルダーにとっての意味が伝わるか？
- [ ] 読み手が企画の全体像を理解できるか？
- [ ] ストーリー・デザインへの展開の土台になっているか？

## ストーリー

### ストーリーの位置づけ

ストーリーは価値デザインモデルの構成要素の一つで、**ビジョンを実現するまでのステップ**を示します。物語的な表現とは異なります。

| 観点 | 説明 |
|-----|------|
| 位置づけ | 価値デザインモデルの「時間軸」を担う要素 |
| 役割 | ビジョン実現までの変化の道筋を描く |
| 対象 | ステークホルダー全体 |
| 入力 | ビジョン、コンセプト（3つ）、言葉、意味 |

### ストーリーの役割

| 役割 | 説明 |
|------|------|
| **変化の可視化** | 現状の課題からビジョン実現後の姿までの変化を具体的に描く |
| **実現性の提示** | 抽象的なビジョンが「実現できそうだ」と感じさせる |
| **行動の動機づけ** | ストーリーを読んだ人が「参加したい」と思えるようにする |

### ストーリーの例
以下のように、ステップを記述する（章立てのイメージ）。

1. 「摩擦からの解放」: 煩雑な紙の手続きや二重入力から解放され、人事担当者は「処理する人」から「戦略を考える人」へ進化。
2. 「成長の可視化」: 評価・目標管理・1on1記録が一元化され、従業員は自分の成長の軌跡を振り返り次の目標を設定できる。
3. 「知恵ある意思決定」: 組織全体の人材データが統合・可視化され、データに基づいた人材戦略を立案できる。
4. 「Well-Beingの実現」: すべてのステークホルダーが本質的な価値創造に集中でき、働く人々が成長を実感しやりがいを持って働ける好循環が生まれる。

## デザイン

### デザインとは
価値デザインモデルの構成要素の一つで、**ビジョンを視覚的に表現するデザインコンセプト**を定めるものです。

| 観点 | 説明 |
|-----|------|
| 位置づけ | 価値デザインモデルの「視覚表現」を担う要素 |
| 役割 | ビジョンの世界観を色・形・イメージで具現化する |
| 対象 | すべてのステークホルダー（特に初見の人） |
| 入力 | ビジョン、コンセプト（3つ）、言葉、意味、ストーリー |

### デザインの役割

| 役割 | 説明 |
|------|------|
| **世界観の視覚化** | ビジョンが描く未来を、視覚的に表現する |
| **第一印象の形成** | ステークホルダーが最初に受ける視覚的印象を決める |
| **ブランドの統一** | 色・形・フォントなどで企画全体の一貫性を担保する |
| **感情への訴求** | 言葉だけでは伝えきれない感情的価値を視覚で補完する |

### デザインの構成要素

| 要素 | 内容 | 問い |
|------|------|------|
| **カラーパレット** | 主要色・補助色・アクセント色 | ビジョンの世界観を表す色は？ |
| **形態・モチーフ** | 丸み・直線・有機的・幾何学的 | ビジョンの印象を表す形は？ |
| **イメージの方向性** | 温かい・先進的・信頼感・親しみやすい | どんな感情を呼び起こしたいか？ |
| **ロゴコンセプト** | シンボルマーク・ロゴタイプの方針 | 企画の本質を一目で伝えるには？ |
| **ビジュアルトーン** | 写真・イラスト・アイコンの使い方 | どんなビジュアル表現が適切か？ |

### デザインの作成方法

**Step 1: ビジョン・コンセプトから視覚的キーワードを抽出する**
- ビジョンが持つイメージ（例: 開放的、温かい、革新的）
- コンセプトが示す方向性（例: 技術的、人間的、社会的）
- 言葉（キャッチコピー）の印象

**Step 2: カラーパレットを決定する**
- 主要色: ビジョンの本質を表す色
- 補助色: コンセプトの多面性を表す色
- アクセント色: 注意を引く、行動を促す色

**Step 3: 形態・モチーフの方向性を定める**
- ビジョンの世界観に合う形の選定
- ステークホルダーに与えたい印象との整合

**Step 4: ロゴコンセプトを策定する**
- ビジョンの本質を視覚的に凝縮する
- シンプルで記憶に残るデザイン

### 良いデザイン・悪いデザインの例

**良い例**:
| 特徴 | 説明 |
|------|------|
| ビジョンとの一致 | デザインを見れば企画の方向性が伝わる |
| 一貫性がある | 色・形・フォントが統一されている |
| ステークホルダーに適切 | 対象者に合ったトーン（例: BtoBなら信頼感） |
| ストーリーと連動 | デザインがストーリーの世界観を視覚化している |

**悪い例**:
| 問題パターン | 問題点 |
|-------------|--------|
| ビジョンと無関係 | おしゃれだが企画の方向性と関係がない |
| トーンの不一致 | 温かいビジョンに冷たいデザイン |
| 複雑すぎる | 要素が多すぎて本質が見えない |
| 対象者への配慮不足 | ターゲット層に合わないデザイン |

### デザインの検証チェックリスト

- [ ] ビジョンの世界観を視覚的に表現しているか？
- [ ] 3つのコンセプトの方向性と矛盾しないか？
- [ ] 言葉（キャッチコピー）の印象と一致しているか？
- [ ] ストーリーの世界観を視覚化しているか？
- [ ] ステークホルダーに適切な印象を与えるか？
- [ ] 色・形・フォントに一貫性があるか？
- [ ] シンプルで記憶に残るか？
