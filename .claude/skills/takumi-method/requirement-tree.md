# Phase 4: 要求分析ツリー

## 要求分析ツリー

### 要求分析ツリーとは

企画を実現するための要求を階層化したモデル。実現可能なレベルまで具体化する。「知」のモデル。

### 要求分析ツリーの構成要素

- 戦略要求（Strategic Requirement）
- 業務要求(Business Requirement)
- IT要求（IT Requirement）
- 活動（Activity）

### 階層構造

- 戦略要求(ビジョン-コンセプト（3個）-目的)-業務要求
- 業務要求以下の構造
    - IT要求と活動は業務要求の下位要求
    - 活動はIT要求の下位要求ではない

```
ビジョン
  └→ コンセプト（3個）
       └→ 目的
            └→ 業務要求
                 ├→ IT要求
                 └→ 活動
```

**重要ポイント**:
- ビジョンが最上位（ルート）
- 左から右へ階層が深くなる
- IT要求と活動は同じレベル（業務要求の下位）
- **IT要求と活動は関連線でつながらない**

業務要求以下の構造:
- IT要求と活動は業務要求の下位要求
- 活動はIT要求の下位要求ではない

---

### 要素間の関係ルール

| 関係 | 可否 | 説明 |
|-----|------|------|
| 業務要求 → IT要求 | ○ | 業務要求をIT化 |
| 業務要求 → 活動 | ○ | 業務要求を人が実行 |
| IT要求 ↔ 活動 | **×** | つながらない（同じレイヤーの独立した選択肢） |
| 同レイヤー同士 | × | 関連線不可 |

---

### 視座レベルの比較

| レベル | 視座 | 例 |
|-------|------|-----|
| 目的 | 経営・事業レベル | 顧客満足度の向上 |
| 業務要求 | 課長・リーダーレベル | 顧客対応プロセスの改善 |
| IT要求 | システム・機能レベル | 顧客管理システム |
| 活動 | 現場作業レベル | 顧客フォローアップ活動 |

## 戦略要求（Strategic Requirement）

- 経営、事業レベルの要求
- 新たに作成するのではなく、他のモデルの内容を機械的に配置する
  - 第1階層(ルート要素)：価値デザインモデルのビジョンを使用する
  - 第2階層：価値デザインモデルのコンセプトを使用する
  - 第3階層：価値分析モデルの目的を使用する

## 業務要求
- 目的(Objective)を達成する手段である（目的の下位要求）
- 経営・事業レベルの要求を実現するための「現場レベルの要求」
- 現場に対して指示する立場の人（課長やリーダー）の視座の要求であり、IT要求（システムやソフトウェアの要求）や、活動よりも抽象的な要求
  - したがって、業務要求の名前が具体的なITの機能やシステムを現す名前の場合、具体的過ぎると判断できる
- IT要求の上位要求である
- 活動の上位要求である
- 業務要求は「価値記述」の「手段」の部分を業務要求として展開したもの

### 業務要求の作成方法
- 価値記述の「手段」部分から業務要求を導出する
    - 価値記述の基本形式
        - 自然言語で「価値を実現する手段」+「価値」の一文で記述する。
        - 「価値」の部分は、必ず **「〜で嬉しい」** という形で表現する
        - 価値記述には、端的に意味を表す名前をつける
    - 価値記述の例
        - **価値記述名**: アイデア発信
        - **価値記述**: 「自分のデザインした家具をマーケットプレイスで公開することで、自分のアイデアを世界に発信できて嬉しい」
            - 価値を実現する手段: 「自分のデザインした家具をマーケットプレイスで公開することで、」
            - 価値: 「自分のアイデアを世界に発信できて嬉しい」
- **【重要】全ての価値記述に対応する、業務要求を作成する**
- 業務要求と価値記述は、基本的には1対1の関係である。

### 業務要求の作成例

**例1:**
- 価値記述: 自分の家具デザインをデジタルプラットフォームで紹介できることで、世界中に自分の作品を広めることができて嬉しい。
- **業務要求名**: 家具デザインをデジタルプラットフォームで紹介

**例2:**
- 価値記述: 仕事を通じて環境に配慮したデザインを推進できるので、サステナビリティに貢献できていると感じられて嬉しい
- **業務要求名**: 環境配慮デザインの推進

**例3:**
- 価値記述: デジタルプラットフォームから安定した仕事の依頼を受けられるので、安定して売上が上がって嬉しい
- **業務要求名**: 安定した仕事の依頼

### 業務要求の良い例・悪い例

**悪い例**:

| 悪い例 | 問題点 |
|-------|--------|
| 「家具デザイン公開機能」 | 具体的すぎる（IT要求レベル） |
| 「プロフィールを登録する」 | 具体的すぎる（活動レベル） |
| 「顧客満足度の向上」 | 抽象的すぎる（目的レベル） |
| 「売上アップ」 | 抽象的すぎる |

### 業務要求の命名ルール

- 動詞＋名詞の形式が望ましい
- 「〜の推進」「〜の実現」「〜の改善」などの表現
- IT機能名（〜機能、〜システム）は避ける
- 抽象的すぎる名前は避ける

## IT要求

- 業務要求を実現するための具体的な手段である（業務要求の下位要求）
- ITシステムやソフトウェアの機能を表す
- システム開発の要件定義に直結する要求

### IT要求の作成方法
- 業務要求の「手段」としてIT機能を導出する
- 各業務要求に対してIT要求を必ず作成する必要はない（無くても良い）
- システムとして実装可能な具体的なレベルで記述する
- 機能名は「〜機能」「〜システム」など明確な名称にする

### IT要求の作成例

**例1:**
- 業務要求: 家具デザインをデジタルプラットフォームで紹介
- **IT要求名**: 家具デザイン公開機能

**例2:**
- 業務要求: 環境配慮デザインの推進
- **IT要求名**: サステナビリティ評価表示機能

**例3:**
- 業務要求: 安定した仕事の依頼
- **IT要求名**: マッチング通知機能

### IT要求の良い例・悪い例

**悪い例**:

| 悪い例 | 問題点 |
|-------|--------|
| 「顧客対応プロセスの改善」 | 抽象的すぎる（業務要求レベル） |
| 「データベース」 | 技術的すぎる、機能が不明確 |
| 「ボタンをクリックする」 | 具体的すぎる（UI操作レベル） |
| 「売上向上機能」 | 抽象的すぎる、実装イメージが湧かない |

### IT要求の命名ルール

- 「〜機能」「〜システム」など明確な名称にする
- システムとして実装可能なイメージが湧く名前
- 業務要求名をそのまま使わない
- 技術用語のみの名前は避ける（例：「API」「DB」だけでは不十分）

## 活動

- 業務要求を実現するための具体的な手段である（業務要求の下位要求）
- 人が行う実際の行動・業務プロセスを表す
- IT要求とは異なり、システムではなく人の活動に焦点を当てる
- システム、ソフトウェアに対する作業は活動の対象外。対象外の例「システムにプロフィールデータを入力する」

### 活動の対象外

以下のような内容はシステム操作であり、活動の対象外とする：
- システムへの登録・編集操作（例：プロフィールを登録する、情報を編集する）
- システム画面の閲覧操作（例：プロフィールを閲覧する、一覧を確認する）
- システムへのログイン・ログアウト
- システムの自動処理

活動は、ユーザーがビジネス上の目的を達成するための具体的な行動を表す。

### 活動の作成方法
- 業務要求の「手段」として人の活動を導出する
- 各業務要求に対して活動を必ず作成する必要はない（無くても良い）
- 人が実行可能な具体的なレベルで記述する
- 活動名は「〜作業」「〜活動」「〜業務」など明確な名称にする

### 活動の作成例

**例1:**
- 業務要求: 家具デザインをデジタルプラットフォームで紹介
- **活動名**: デザイン作品の撮影、紹介ページ作成作業

**例2:**
- 業務要求: 環境配慮デザインの推進
- **活動名**: サステナビリティ基準の策定・レビュー

**例3:**
- 業務要求: 安定した仕事の依頼
- **活動名**: 顧客対応・フォローアップ活動

### 活動の悪い例

**悪い例**:

| 悪い例 | 問題点 |
|-------|--------|
| 「顧客対応プロセスの改善」 | 抽象的すぎる（業務要求レベル） |
| 「システムにプロフィールを登録する」 | システム操作（活動の対象外） |
| 「プロフィールを閲覧する」 | システム操作（活動の対象外） |
| 「データを入力する」 | システム操作（活動の対象外） |
| 「売上向上」 | 抽象的すぎる、行動が不明確 |

### 活動の命名ルール

- 「〜作業」「〜活動」「〜業務」など明確な名称にする
- 人が実行可能なイメージが湧く名前
- 業務要求名をそのまま使わない
- システム操作を含めない
