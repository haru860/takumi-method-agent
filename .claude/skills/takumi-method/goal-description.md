# ゴール記述モデル(goal-description)

## ゴール記述モデル
### ゴール記述モデルとは

MSP計画の作業項目を、具体的なゴール記述として細分化し、
誰が・何を・いつまでに・どのように達成するかを明確にする。

構想を実行可能なアクションプランに落とし込み、進捗を測定可能にする。

---

### ゴール記述モデルの構成要素

ゴール記述モデルは、以下の8つの項目で構成される。

| 項目 | 説明 | 例 |
|------|------|------|
| テーマ | 何をすることで（上位目標・Phase名） | Phase 1: 基盤整備 |
| 役割名 | 誰が（担当者の役割） | 開発チーム |
| 何を | 対象となる成果物・機能 | 匠Method Agentの研修用環境 |
| 開始時期 | いつから（具体的な日付・時期） | 2026年3月1日 |
| 終了時期 | いつまで（具体的な日付・時期） | 2026年3月31日 |
| どうする | 具体的なアクション | セットアップする |
| 評価尺度 | 何で測るか | 環境稼働状況 |
| 目標値 | どのレベルを目指すか | 受講者が演習できる環境が稼働 |

### 品質基準

- **具体性**: 「何を」「どうする」が具体的で、実行イメージが湧くこと
- **測定可能性**: 評価尺度と目標値が定量的・客観的に測定可能であること
- **スケジュールの現実性**: 開始時期・終了時期が現実的で、依存関係が考慮されていること
- **役割の明確さ**: 役割名が具体的で、誰が担当するかが明確であること
- **網羅性**: MSP計画のすべてのPhase・作業項目がカバーされていること
- **粒度の適切さ**: ゴール記述の粒度が大きすぎず細かすぎず、管理可能であること
- **MSP計画との整合**: MSP計画の作業項目と対応関係が明確であること
- **ビジョン・コンセプトとの整合**: 上位の価値設計と一貫したゴール設定であること

### 良い例・悪い例

### 良い例

| 何を | どうする | 評価尺度 | 目標値 |
|------|----------|----------|--------|
| 研修用環境 | セットアップする | 環境稼働状況 | 受講者が演習できる環境が稼働 |
| 講義資料 | 作成する | 資料完成状況 | 講義スライド一式完成 |

### 悪い例

| 悪い例 | 問題点 |
|-------|--------|
| 「いろいろ整備する」 | 「何を」が曖昧で実行イメージが湧かない |
| 「うまくいくようにする」 | 「どうする」が抽象的 |
| 「良い感じになったら完了」 | 評価尺度・目標値が主観的で測定不能 |
| 「1日で全Phase完了」 | スケジュールが非現実的 |

## ゴール記述の作成方法

### MSP計画からの細分化ルール

1. **テーマ**: MSP計画のPhase名を使用
2. **役割名**: MSP計画の「担当」列を使用（なければ推定）
3. **何を**: 作業項目から対象物を抽出
4. **開始時期**: プロジェクト開始時期を基準に、Phaseと依存関係を考慮して算出
5. **終了時期**: Phase目安期間と依存関係を考慮して算出
6. **どうする**: 作業項目から動詞を抽出
7. **評価尺度**: 完了条件から測定可能な指標を抽出
8. **目標値**: 完了条件をそのまま、または具体化
