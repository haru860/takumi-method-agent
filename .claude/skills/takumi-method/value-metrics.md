# 匠Method Value Metrics（VM）

## 匠Method Value Metrics
### 匠Method Value Metricsとは

企画の価値を定量的に評価し、価値と実現手段の整合性を検証するための手法です。

### 匠Methodのプロセスとの関係
ステップバイステップで進める場合、要求分析ツリーの目的まで作成した時点でステップ1（`/vm-value-concept-score`）を実行できます。ステップ2以降はIT要求・活動の作成後に実行します。

### Value Metrics の3つのステップ

| ステップ | 内容 | 成果物 |
|---------|------|--------|
| 1. 価値概念の作成とインパクト評価 | 各目的に対して価値概念を作成し、インパクトスコアで評価する | `vm-value-concept.tsv` |
| 2. IT要求・活動のインパクトスコア算出 | IT要求・活動それぞれのインパクトスコアを算出する | `vm-it-requirement-impact-score.tsv`, `vm-activity-impact-score.tsv`, `vm-it-requirement-activity-impact-score.tsv` |
| 3. スコア乖離検証 | 価値概念スコアとIT要求・活動スコアを目的別に突き合わせ、乖離を検証する | `vm-compare-score.md` |

### インパクト評価基準

価値概念、IT要求、活動が実現した時のインパクトを、以下のフィボナッチ数列ベースのスケールで採点します：

| スコア | 評価 | 説明 |
|--------|------|------|
| 1 | 全然ダメ | 価値がほとんど感じられない |
| 3 | いまいち | 価値はあるが弱い |
| 8 | もう一つ | 一定の価値はあるが差別化にならない |
| 13 | ちょっと良い | 明確な価値があり、採用理由になる |
| 21 | かなり良い | 強い価値があり、競合との差別化になる |
| 34 | 大変良い | 非常に高い価値があり、市場で優位に立てる |
| 89 | ダントツ | 業界を変革するレベルの価値、追随困難 |

### 89点（ダントツ）の条件

- **不可逆的な変化**を起こす（一度体験したら戻れない）
- **業界・市場のルールを変える**（ゲームチェンジャー）
- **競合が容易に追随できない**独自性がある
- **社会的インパクト**が明確に測定できる

## 価値概念のスコア付け

### 価値概念とは

各目的に対して「この企画が実現する最も重要な価値」を一文で表現したもの。以下の構文から導き出せる価値のイメージです：

> $ビジョンを実現するために、$コンセプトの取り組みの一環として、$目的に取り組んだ結果、『$価値概念』ようになった。

**$価値概念 = 「〜ができる」** という形式で表現します。

### 価値概念の表現ルール

- 「〜ができる」という形式で表現
- 『』で囲む
- 具体的で測定可能な表現を心がける
- ステークホルダーにとっての価値が明確に伝わる表現にする

### 価値概念の評価観点

- 市場での差別化になるか
- ステークホルダーの課題解決度
- 競合が追随困難か
- 経済的インパクトの大きさ
- 社会的意義

## IT要求、活動のスコア付け

### IT要求のインパクト評価観点

- IT要求が実現した時の価値の大きさ
- システム化による効率化・自動化のインパクト
- ステークホルダーの課題解決度
- 市場での差別化になるか
- 競合が追随困難か
- 経済的インパクトの大きさ

### 活動のインパクト評価観点

- 活動が実行された時の価値の大きさ
- ステークホルダー体験の向上度
- ステークホルダーの課題解決度
- 市場での差別化になるか
- 競合が追随困難か
- 経済的インパクトの大きさ

## スコア乖離検証

### スコア乖離検証とは

価値概念（vm-value-concept.tsv）のスコアと、IT要求・活動（vm-it-requirement-activity-impact-score.tsv）のスコアを目的別に突き合わせ、乖離がないかを検証するもの。乖離がある場合、価値を実現する手段（IT要求・活動）に過不足がある可能性を示す。

### 比較方法

1. `vm-value-concept.tsv` から目的名、価値概念、スコアを取得
2. `vm-it-requirement-activity-impact-score.tsv` から目的別にIT要求・活動をグループ化
3. 同じ目的に紐づくデータを突き合わせる
4. 以下の観点で乖離を検証：
   - 価値概念のスコアが高いのに、対応するIT要求・活動のスコアが低い
   - 価値概念のスコアが低いのに、対応するIT要求・活動のスコアが高い
   - 価値概念に対応するIT要求・活動が存在しない

### 乖離の判定基準

| 判定 | 基準 |
|------|------|
| ○（整合） | 価値概念スコアとIT要求・活動スコアの差が20%以内 |
| △（軽微な乖離） | 差が20%〜40% |
| ×（大きな乖離） | 差が40%以上、または対応なし |
